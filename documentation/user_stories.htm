Benetech Martus Amplifier

User Stories

  • user searches for a bulletin that "recently" got added to backup server, and the search returns that result
    • polling
      • when amp starts, it sets up a chron-like timer
      • whenever timer rings (e.g. once a day), amp polls the servers
      • kevin says once per day polling should be adequate
    • [working] amp gets list of servers to sync with
      • get list from some sort of config file set up by admin person who installed amp
    • [sri] amp makes request to server to get list of new bulletins
      • request handled at first just by gateway stub, later by the gateway calling the handler interface
    • [brian] amp gets list of bulletins it already has
    • [brian] amp compares the new list to the old list of what it already has
    • [sri] amp gets new bulletins (and attachments) from server
      • gateway does decryption and unzipping
    • [dan] amp indexes new documents
    • [dan] amp deals with user any access concurrency problems during re-indexing
  • search page
    • [working] user types URL in browser, browser returns amp search page
    • [working] amp search page has a text field and a search button
    • [working] user hits search button with search-text field empty, browser returns something
    • [in progress] Have a checkbox for case sensitive/insensitive (default should be insensitive)
    • [in progress] Have a checkbox for whole word/partial word (default should be partial word)
    • [in progress] Have a different search page that allows searching a date range in EventDate
    • [in progress] User specifies which field(s) to search (using a multi-select dropdown)
  • contact page
    • [initial version working] user reports bug to Benetech
  • faq page
    • [initial version working] user sees list of questions and answers
  • result list page
    • [working] Display search results in a table
    • [working] results table has exactly one row for each search result
    • [working] results table has one column for each field: Author, EventDate, Title
    • [working] results table has one bulletin-link in each row
    • [working] user clicks on bulletin-link in results tables, browser goes to bulletin page
    • [sri] user can sort by column (e.g. author, title, entry date) by clicking on the field title
    • [working] if results don't fit on one page, results are split between pages
    • [working?] user prints results page (using browser print button)
    • [working?] user sends copy of results page in e-mail (by copy & paste?)
  • bulletin page 
    • [working] bulletin page is a single-page view of all the fields in that bulletin
    • [dan] bulletin page shows attachments
    • [working] Add a "next" button to the single bulletin view page that takes you to the next match
    • [working] Add a "previous" button to the single bulletin view page
    • user clicks on author link to go to author page
    • [working?] user prints results page (using browser print button)
    • [working?] user sends copy of bulletin page in e-mail (by copy & paste?)
  • author page
    • author page is a single-page view of all the "contact info" available for an author
  • error page (error.jsp?)
    • [sri] user sees clear message if back-up server is down
    • [sri] user sees clear message if search returned no results
